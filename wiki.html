Compiling it

You will first have to get the source on the Github repo

Now open a terminal and cd to the directory where you cloned/extracted the source code

Now to compile it follow these commands:


GCC:
gcc src/easymake.c -Iinclude -s -O3 -o easymake

TCC:
tcc src/easymake.c -Iinclude -O3 -o easymake

CLANG:
clang src/easymake.c -Iinclude -O3 -o easymake

MSVC:
cl src/easymake.c /I include /O2

You can add Easymake to your environment variables to access it from anywhere on your system.


Your first build file

Once you've compiled Easymake, you're ready to start migrating your project!

Easymake build files are JSON files with the .ezmk extension. Anyone familiar with basic JSON can write one with ease.

The default build file should be titles build.ezmk. If you opt to use a different name, you must pass the -f [filename] argument.


{
  "project": "easymake",
  "compiler": "gcc",
  "output": "easymake",
  "includes": ["include"],
  "sources": ["src/easymake.c"],
  "compiler_options": ["-s", "-O3"]
}

To execute the build file, simply run $ easymake. This will output your binary to the output location.

If you've opted to use a different filename, run $ easymake -f [filename].ezmk, which will result in the same output.


More about build files

On includes, sources and compiler options you can also do a array

like you do on the compiler_options of the example

heres an example:


{
  "project": "easymake",
  "compiler": "gcc",
  "output": "easymake",
  "includes": ["include", "path/to/includes"],
  "sources": ["src/easymake.c", "path/to/source/file.c"],
  "compiler_options": ["-s", "-O3"]
}

What are these values in the build file?

project

This is the title of your project. As of now, it is simply for organizational purposed, but it may be used later.

Example:


"project": "project name"

compiler

This is your compiler, the initial command Easymake will run. This is not limited to just C either!
This should work with any compilers, such as TCC, MSVC, or even other languages.

Example:


"compiler": "g++"

output

This is the name of the compiled executable. It can contain a small folder structure as well for organization.

Example:


"output": "bin/output"

includes

This is your include path. Must be an object array, but can contain just one path if required.

Example:


"includes": ["include","/usr/include/", "path/to/idk/includes/i/guess/"]

sources

The source files to be compiled. Must be an object array, but can only contain one file if required.

You can also do a entire folder by writing folder/*.[ext]

Example:


"sources": ["src/main.cpp", "src2/*.cpp"]

compiler_options

Extra compiler flags must be passed as an object array, but can only contain one if needed.

Example:


"compiler_options": ["-Wall"]
